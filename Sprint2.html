Class Wars: ID vs Class

In this post I will first briefly dicsuss the differences between Class and ID and then outline some of the best practices you should consider when using these two identifiers in your work.

What is an Identifier?

Both Class and ID can be considered an identifier. You can use them when you need to refer to a specific element within your hmtl code from within your CSS file. But what is the difference bewteen them, and when should I use ID as opposed to class?

All part of the Team

The most important destinction between ID and Class is that of scope. ID is very specific and there can only ever be one instance of a particular ID in your code. Class is much more flexible and can be applied to many different elements within the same page, and an element can even have mulitple classes applied to it. A real life analogy can be found in the murky world of sportsball. If we think about a stock standard team of sportsballers, the position a player plays in will be its ID. There may be lots of players running around, but there can only ever be one Boss Ballstopper per team. Class would be more akin to the Team in the sense that all members are part of the same entity and that they all wear the team jersey regardless of where they play. This analogy can also be extended further as any element on a page can have more than one class assigned to it - just as a sportsballer can play different kinds of sport while still being a member of the team.

Best Practices

In most cases it would be preferable to use Class rather than ID. This is mainly due to the incredible flexibility of a class. You can set them up once and then apply them across your entire site, fine tuning and applying multiple levels of classes to get your job done quickly and effectively across all elements sharing the assigned class.

IDs trade their flexibility in order to be referenced directly. This specificity allows you to fine tune your elements to very fine degree but only ever the one element that bears that ID. IDs also outrank classes, and can be used to overwrite any styling stemming from a class. In short, due to being unique IDs excel at being called and referred to by workflow elements such as direct styling, anchor links and is particulary handy in Javascript.

To summarise - when you need to apply styling across a range of elements without creating referential chaos, use Class. When it comes to styling one very particular element and that element only, or if you need to refer or call an object directly, use ID.
